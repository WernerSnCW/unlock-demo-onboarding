Build Brief — Syndication (Free Version)
0) Purpose (one line)

Help investors discover active syndicates, see anonymised confidence signals (“investors like you are short-listing this”), request to join (nominal fee: stub), and preview an AI Syndicate Builder Assistant (locked).

1) Routes

/syndication — discovery index (all open syndicates).

/syndication/:id — syndicate detail for one company.

Entry points:

From Business Profile → Community → Join Discussion.

From Business Directory cards (new chip: Open Syndicate).

2) Page structure
A) /syndication (Discovery)

Header: “Syndication” + subline “Explore open syndicates and community confidence signals.”

Filters row: Sector (chips), Stage, Eligibility (EIS/SEIS), Min cheque band, Closing window (≤ 2 weeks / 30 days / 90 days).

Cards grid (3-across desktop, 1-col mobile). Each SyndicateCard shows:

Company name + ✅ Verified chip + sector chips.

Raise target (e.g., “Target £1.2m”), Committed bar with %.

“Investors like you” badge (e.g., “21% of angels like you shortlisted this”).

Confidence meter (0–100) with tooltip.

Meta row: Lead (alias) · Min cheque · Closes in 18d · EIS/SEIS chips.

CTA: View syndicate (→ /syndication/:id).

B) /syndication/:id (Detail)

Layout: two columns

Left (main)

Overview card: raise target, committed %, min cheque, closing date, carry/fees (if disclosed), EIS/SEIS eligibility.

Confidence & Signals card (see §5).

Activity Timeline (last 10 events): Q&A posted, commitments, updates.

Right (actions & preview)

Join Syndicate (Nominal Fee) card (stub): “Reserve a seat — £25” → modal (no payments; just a success toast + badge “Requested”).

Syndicate Builder Assistant — Preview (locked): blurred sample outreach draft + Upgrade to Pro CTA.

FAQs / Risks: clear “This is not advice. Community interest is not a recommendation.”

5) “Investors like you” & Confidence — simple model (front-end)
A) Like-You % (badge)

Compute overlap with each member in the syndicate cohort (mocked).

Similarity score per member:

+0.5 if sector intersects

+0.3 if ticket band matches

+0.2 if risk ±1 band

“Like-You” cohort = members with score ≥ 0.7.

likeYouPct = likeYouCount / totalMembers × 100 (we’ll store this in interest.likeYouPct for now).

B) Confidence Score (0–100)

Weighted blend (store as interest.confidenceScore for prototype):

Commitment momentum (last 3 deltas from history) → 40%

Lead credibility (static 0–100; derive from lead.trackRecord) → 30%

Overall interest (overallPct) → 30%

Show as meter with bands: 0–49 Low, 50–74 Moderate, 75–100 High.

Tooltip: “Composite of momentum, lead track record, and community interest. Informational, not advice.”

Committed tooltip: “% of target raise soft-circled by the group.”

Nominal fee footnote: “Reservation fee shown for illustration. No transaction occurs in this prototype.”

9) Styling (tokens only)

Use brand accent for meters/badges; shadows subtle; cards radius 12px.

Confidence meter: gradient from muted → amber → success.

Cohort bars (Angels/Funds/FO): tiny stacked bars with labels.

Sparkline: simple SVG polyline; no external chart lib required.

10) Behaviours

Filters combine with AND; text search matches company + lead alias.

Discovery cards lazy-load logos (placeholder avatar if none).

Skeletons on first load and when changing filters.

On /syndication/:id, requested state persists in local storage.

Accessibility:

All controls tab-reachable; visible focus ring.

Meter has aria-valuenow with text alt.

Announce toasts via aria-live="polite".

11) Acceptance criteria (Definition of Done)

Discovery page lists syndicates with: verified badge, raise target, committed bar, Investors like you badge, confidence meter, meta row, and View CTA.

Filters (sector, stage, eligibility, cheque band, closing window) work together; search works.

Detail page shows overview, Confidence & Signals, Activity Timeline, Join Syndicate (stub), and Assistant Preview (locked).

“Request to Join” opens modal; submitting sets “Requested” state and shows success toast (no payments).

Confidence meter and Investors like you render with tooltips; cohorts and sparkline display.

All UI uses CSS variables; AA contrast; keyboard navigation passes a basic check; no console errors.
