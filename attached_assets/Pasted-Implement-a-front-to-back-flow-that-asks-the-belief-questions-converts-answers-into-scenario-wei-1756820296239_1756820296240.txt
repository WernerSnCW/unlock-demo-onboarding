Implement a front-to-back flow that:

asks the belief questions,

converts answers into scenario weights and picks/blends the top scenario(s),

pulls the Base allocation for that scenario,

applies persona constraints to produce a Persona-Adjusted Example Portfolio, and

displays the result with a short rationale.

Inputs (use these modules exactly)

Belief questions & mapping → scenarios: questions.py, beliefs.py, beliefs_ml.py. These contain question text, option→scenario weights, and the optional ML overlay to turn answers into S001–S010 scenario weights. 
 

Scenario Base allocations: defaults.py (exposes scenario default mixes you start from once a scenario is chosen). (Referenced in the execution guide as SCENARIO_DEFAULTS.) 

Persona traits: persona_defs.py (liquidity months, drawdown cap, property/tech/alternatives bias, concentration tolerance, income tilt). 

Persona rules (no optimisation): rules.py or persona_rules_and_cvar.py, specifically apply_persona_rules(base_alloc, persona, scen_id) to enforce floors/caps and edge cases. Do not call any CVaR/LP functions. 

Functional requirements
A) Belief → Scenario selection

Render a short beliefs quiz (7–12 questions) sourced from questions.py (use the items already carrying scenarios:{...} weights such as q10, q12–q16, q24–q26). Sum the scenario nudges per answer to form raw scores for the ten scenarios. 
 

Normalise raw scores to weights with temperature-scaled softmax (defaults):

meanCenterScores = true, softmaxTemperature = 0.5, maskedSoftmaxThreshold = 0.001. (This prevents the “all ~10–15%” flatness.) 

Rank scenarios; apply decision rules:

Clear winner: Top − #2 ≥ 10 pts → use Top.

Close: 3–9 pts → blend Top/#2 (gap 9–7→65/35; 6–4→60/40; 3→55/45).

Indecisive: Top − #2 < 3 → fall back to Neutral/Base and prompt to refine later.
(Display as “match to your answers”, not probabilities.)

B) Scenario ID mapping

Create a tiny alias map so belief outputs like "reflation","devaluation","energy_spike" resolve to S006, S008, S010 (etc.) for the defaults/rules step. Keep a 1-to-1 mapping table (strings → S00x). Use the scenario library names as reference for the correct S-codes. 
 

C) Build the Base mix

If a single scenario won: base = SCENARIO_DEFAULTS[Sxx].

If blending two: base = w1*Base(S_top) + w2*Base(S_2nd).
(Use the defaults.py object that exposes scenario defaults.) 

D) Apply persona constraints → Persona-Adjusted Example

Read the chosen persona object from PERSONAS[pid] in persona_defs.py. 

Call only: apply_persona_rules(base, persona, scen_id) from persona_rules_and_cvar.py (or rules.py).

Must enforce: minimum liquidity floor from persona’s liquidity_months, concentration caps, property/crypto special cases (e.g., P016 property caps; P003 crypto cap under S004), and re-normalise to 1.0. Do not run CVaR optimisation or Monte Carlo. 
 

UI/UX requirements

Step 1 – Questions: progress bar, concise UK-English prompts, and a “Skip for now” that will default to Neutral/Base.

Step 2 – Scenario result: show a horizontal bar chart of the top 5 scenario matches; under it, a banner:

“Using Reflation” or “Blending Reflation / Energy Spike 60/40”.

Step 3 – Portfolio cards (two columns on desktop):

Base (Scenario) — table + mini bar chart of weights.

Persona-Adjusted (Example) — after rules applied. This is the one you highlight as the example portfolio.

Why this mix: two bullet lines:

Scenario tilt summary (e.g., “Energy Spike → higher gold/commodities, less duration”). 

Persona guardrails (e.g., “P016 cap on property; liquidity floor raised to 12%”). 

Footer note: “Illustrative example based on your persona and answers; not advice.”

Non-goals (explicitly out for now)

No simulations (no Monte Carlo, no CVaR LP). Ignore risk.py, engine.py, optimise_cvar_ru.py. 

Validation & edge cases

If no belief answers provided → pick Neutral/Base scenario and still apply persona rules (liquidity floor, caps). 

Ensure the weights sum to 100% after rules (re-normalise).

Enforce liquidity floor = max(10%, 2% × liquidity_months) across Cash + Short Gilts as implemented in rules. 

For P016 in S009/S007 ensure the lower property cap kicks in before display. 

For P003 in S004 ensure the crypto cap and reallocation to IG/Gilts is applied. 

Deliverables

A callable that returns:

scenario_selection: {top: "S00x", runner_up: "S00y"|null, blend: [w1,w2]}

base_allocation: {asset: weight}

persona_adjusted_allocation: {asset: weight}

explanations: {scenario_reason: "...", persona_rules_applied: ["...","..."]}

A single screen that renders the above with copy in UK English.
