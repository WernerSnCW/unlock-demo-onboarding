CSV & Manual (distinct fallbacks)

CSV: User uploads a broker export, maps fields, imports holdings → Source = ⤿ Semi-auto (editable Units/Cost).

Manual: User enters ISIN/ticker, Units, Cost/Price, Dates → Source = ✍ Manual (fully editable).

Legend chips above the table:

⚡ Live — Positions & prices update automatically.

⤿ Semi-auto — Prices auto; positions from CSV/manual.

✍ Manual — Entered by you; attach evidence.

Edge cases Replit should handle (UI)

Merge flow: When a Live holding matches an existing row, default the action to Merge (no duplicate).

Zero/closed positions: Show but default to Ignore.

No book cost from broker: allow a one-time Cost basis field during import (marked “user supplied”) but keep Source = Live.

Partial coverage: If a broker doesn’t return all accounts, let the user re-open Pick Accounts from a link under the table title.

Acceptance criteria (so “done” is unambiguous)

“Add → Listed security” starts with Source selector; Connect broker leads to Broker Picker → Consent → Auth (mock) → Pick Accounts → Review & Pick Holdings.

Import creates/updates rows with Source = Live, locked Units/Cost, and Provenance in the drawer.

CSV and Manual paths are present and clearly different; rows show ⤿ and ✍ respectively.

A Source legend appears above the register; a Source column appears in the table.

“Change source” link returns the user to the source selector.

No duplicates after import when Merge is used.
