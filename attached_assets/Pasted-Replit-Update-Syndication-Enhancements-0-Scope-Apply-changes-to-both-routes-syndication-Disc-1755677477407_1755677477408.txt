6) Components behaviour
FeeBreakdown

Props: { fee: { carryPct, adminAnnualPct, platformFeeGBP } }

Small inline view + modal with example cost table:

“If total gains £X, 10% carry = £Y” (static example).

WhyThisSyndicate

Props: { thesis: string[], risks: string[], sources?: string[] }

Collapsible on mobile; bullets capped to 5 each; “See more” toggle.

TrustBadges

Props: { leadVetted: boolean; governanceAudited: boolean; docPackAvailable: boolean }

AA-contrast chips; show tooltip copy (paste):

Lead Vetted: “Lead identity and track record reviewed by Unlock.”

Governance Audited: “Key governance controls independently reviewed.”

Docs Available: “Lead memo and term sheet available for review.”

FractionalOwnership (preview)

Props: { unitName, unitsOffered, unitPriceGBP, minUnitsPerInvestor }

Disabled input for quantity + disabled CTA; helper: “Secondary market/liquidity under exploration.”

BenchmarkPanel

Renders comparison bars + tiny table of comps.

BundleCard

Props from bundle mocks; show confidence, diversification, min cheque, target raise; CTA open detail.

10) Definition of Done (QA)

Discovery shows enriched cards (WhyThis popover, Fee badge, Trust badges) and a Bundles tab with working BundleCards.

Sort includes “Lowest Fees” and “Highest Confidence”.

Detail page displays FeeBreakdown, WhyThisSyndicate, BenchmarkPanel, FractionalOwnership (preview), and existing sections without layout breakage.

Doc pack stubs appear when docPackAvailable=true.

Bundles modal shows weights and included syndicates; CTA present (locked).

All components use CSS variables only; AA contrast; keyboard accessible; no console errors.

Implementation notes

Reuse your existing chart style (sparklines/progress). Benchmarks can be simple CSS bars; no new chart lib required.

Keep all numbers formatted with a helper formatGBP() and formatPct() for consistency.

Use your existing toasts for “preview/locked” interactions.
