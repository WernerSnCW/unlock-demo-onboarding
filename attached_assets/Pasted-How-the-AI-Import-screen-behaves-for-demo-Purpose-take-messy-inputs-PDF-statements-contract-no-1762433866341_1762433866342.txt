How the AI Import screen behaves (for demo)

Purpose: take messy inputs (PDF statements, contract notes, CSVs, screenshots) and propose clean actions: link evidence, create transactions, update holdings, or ignore.
Flow (3 steps):

Upload → user drops files; we classify each (e.g., “Vanguard ISA statement, Oct ’25”, “Contract note”, “Bank statement”, “Ledger CSV”).

Review & match → for every file we show what we detected, confidence, which account/wrapper it likely covers, and what we’ll create (transactions, holdings updates, or just evidence links).

Import → show a short summary (“Create 12 transactions • Link 4 docs • Update 2 prices”), then Import.

Key affordances:

File pills with type + confidence.

A table of proposed actions per file (Create/Link/Ignore).

Coverage selector (Account/Wrapper/Asset).

A tiny privacy note (“demo: nothing leaves your browser”).

Microcopy (optional, paste as tooltips)

Detected: “We classify the file, extract dates, symbols and amounts, and guess the right account/wrapper.”

Confidence: “Our certainty (0–1). Low confidence items default to Link only.”

Creates: “What we’ll add to your register. Nothing is final until you tap Import.”

Acceptance criteria for the demo

Clicking ✨ AI Import opens the 3-step modal.

Shows three example files with detected types, coverage, and actions.

“Import 3 items” is a non-functional button (for demo only).

Works with your existing dark UI and lightbox styles.
