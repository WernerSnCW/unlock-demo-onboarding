Objective

Turn Transactions into an audit-ready ledger that:

shows all economic events (not just buys/sells),

lets users ingest activity by Live, CSV, OCR from Documents, or Manual,

links every row to Evidence, Account/Wrapper, Holdings, and Cash impact, and

supports reconciliation & cost basis later.

B) Ingestion routes (the “Add / Import” menu)
B1) ⚡ Live (preferred, where available)

Flow: Connect broker (existing) → Pick accounts (existing) → New: Import transactions

Screen: “Import transactions”

Period selector: From/To (default last 90 days).

Table preview with Include/Merge/Ignore.

Columns (read-only from broker): Date, Type, Identifier, Asset, Account, Wrapper (editable), Qty/Nominal/Contracts, Price, Ccy, FX, Gross, Fees, Tax, Net, Broker ref.

CTA: Import n transactions.

Result:

Create rows with Source = ⚡ Live.

Lock fields in the drawer (edit disabled except Wrapper if needed).

Auto-link to the Cash account of the same broker account.

De-dupe rule (apply on import): broker_ref OR (trade_date + type + identifier + account + quantity + net_amount).

B2) ⤿ CSV import

Flow: Upload → Detect template → Map fields → Review → Import

Supported demo templates in dropdown: AJ Bell, Hargreaves Lansdown, interactive investor, Vanguard, Generic.

Mapper must capture: Trade date, Settle date, Type, Identifier, Name, Account, Wrapper, Qty/Nominal/Contracts, Price, Trade ccy, FX, Gross, Fees, Tax, Net, Broker ref.

Normalise signs (e.g., SELL often negative quantity, dividends positive net).

After mapping, show the same Review table & de-dupe logic as Live.

Result rows: Source = ⤿ CSV (editable).

B3) 📎 From Documents (OCR)

Entry points:

From Documents card: “Extract transactions”

From Transactions toolbar: “From a statement/contract note”

Flow: Pick document → Parse preview → Confirm mappings → Import

Link the resulting rows to that document id and mark Evidence = Strong.

Source = 📎 OCR (editable).

B4) ✍ Manual (edge cases)

Small modal with Type first; fields change by type:

Buy/Sell (listed) → Identifier, Name, Account, Wrapper, Quantity, Price, Trade ccy, FX, Gross, Fees, Tax, Net (auto calc allowed), Trade & Settle dates.

Dividend/Interest → Identifier/Name (or Cash account), Account, Wrapper, Net amount, Trade/Pay date, Ccy/FX, Reinvested? (Yes creates a matching Buy).

Fee/Charge/Tax → Account, Wrapper, Amount, Date, Ccy/FX, Fee type (commission, custody, FX, stamp, PTM).

Transfer/Contribution/Withdrawal → From/To account, Amount, Date, Ccy/FX.

Corporate action → subtype (split/consolidation/merger/spin-off/rights/scrip), key parameters.

Bond buy/sell → add Nominal, Clean/Dirty price, Accrued interest.

Derivative event → open/close/exercise/assignment/expiry, Contracts, Strike, Put/Call, Multiplier.

All manual rows: Source = ✍ Manual.

Net amount sign = cash direction for the account.

D) Validation & de-dupe

Dates: settle ≥ trade (except some corporate actions).

FX: required when trade ccy ≠ GBP (CSV/Manual/OCR).

Bond buys/sells: accrued ≥ 0; nominal > 0.

Manual dividends: if “Reinvested”, create a matching Buy (link the pair).

De-dupe: before insert, try broker_ref; if absent, match on (trade_date, type, instrument_id, account_id, quantity|nominal|contracts, net_amount) within ±2 days on settle.

E) Reconciliation hooks

A transaction row is Reconciled = Yes when its settle date falls inside a statement period linked to the same account in the Reconciliation tab and sum of cash deltas matches the statement total.

Show a “Mark period reconciled” action in Reconciliation once:

statement is linked and

∑ (Net amount for period & account) + starting cash = ending cash.
