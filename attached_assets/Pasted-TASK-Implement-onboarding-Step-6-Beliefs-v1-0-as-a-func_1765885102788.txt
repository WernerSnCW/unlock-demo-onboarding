E) Scoring maths (deterministic)

For each answer a (1..5) compute normalised:

n = (a - 3) / 2 (so 1→-1, 2→-0.5, 3→0, 4→0.5, 5→1).

Compute axis scores (clamped [-1, +1]) for 8 axes:

QUALITY_TILT = n(Q_QUALITY)

VALUE_TILT = n(Q_VALUE)

TECH_TILT = n(Q_TECH)

UK_BIAS = n(Q_UK_BIAS)

ESG_TILT = n(Q_ESG)

INFLATION_HEDGE_TILT = n(Q_INFLATION)

SMALL_CAP_TILT = n(Q_SMALL_CAP)

VOLATILITY_AVERSION = -n(Q_VOLATILITY_COMFORT) (inverse)

For each axis score, compute:

direction:

TOWARDS if score ≥ +0.20

AWAY if score ≤ -0.20

NEUTRAL otherwise

intensity using absolute score thresholds:

NEUTRAL: 0.00–0.19

LIGHT: 0.20–0.49

MODERATE: 0.50–0.79

STRONG: 0.80–1.00

Build tilt_profile list (one item per axis) containing:

axis_code, direction, intensity, score, and a neutral description.
Provide axis descriptions (static mapping) such as:

TECH_TILT: “Shifts exposure towards technology relative to a neutral baseline.”

UK_BIAS: “Shifts exposure towards UK-listed assets relative to global neutral.”

VOLATILITY_AVERSION: “Favors smoother risk profiles over higher volatility.”
(and similar short descriptions for the other axes).
