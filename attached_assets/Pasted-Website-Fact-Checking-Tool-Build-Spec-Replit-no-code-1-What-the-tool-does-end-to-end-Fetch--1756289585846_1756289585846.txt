2) Architecture (modules & responsibilities)

PageFetcher: Headless browser fallback for CSR pages; otherwise simple HTTP GET. Throttle; respect robots.txt.

Extractor: Use a boilerplate remover (readability-style) + de-dupe + language detect (en focus) + collect metadata (title, canonical URL, publish date if available).

ClaimMiner (LLM pass #1): Runs the Claim Extraction Prompt (below). Emits structured JSON claims.

Entity Resolver: Map organisation/person/place strings to normalised entities. For UK companies, attempt Companies House disambiguation (by exact name, postcode, or domain in text).

EvidenceFetcher:

News: Build queries per claim (organisation, key terms, time window from text), call NewsAPI, store top N hits with titles, URLs, dates, and snippets. 
News API

Companies House: If a company claim is detected, call /search/companies to get a company_number, then fetch /company/{company_number} and related as needed (status, incorporation date, registered office, SIC, officers). 
developer-specs.company-information.service.gov.uk

Verifier (LLM pass #2): Given a single claim and the collated evidence (articles + CH JSON), produce a verdict, confidence, and rationale (using the Verification Prompt below).

ReportBuilder: Aggregates results and renders UI.

5) Claim selection rules (what’s “in scope”)

Include verifiable factual claims, e.g.:

Dates, counts, amounts (“raised £5m in June 2024”)

Company facts (status, incorporation date, registered address, directors)

Concrete capabilities/coverage (“available in 120 countries”)

Rankings/awards (“winner of 2023 Queen’s Award”)

Market share & performance metrics

Promises/forecasts that reference measurable targets (mark as forecast)

Exclude / mark as opinion (and usually skip):

Pure puffery (“world-class”, “best-in-class”) without measurable anchor

Vague claims without entities/units/timeframe

6) Verification rubric (how to score)

Verified: Matches Companies House (for company facts) OR ≥2 reputable independent NewsAPI sources in agreement; dates/entities match exactly; no credible contradictions found.

Partially verified: Key parts supported but (a) some detail missing, (b) only 1 reputable source corroborates, or (c) timeframe/quantity differs within ±15%.

Contradicted: Strong evidence from CH or ≥2 reputable sources indicates the claim is false/out-of-date.

Unverifiable: No suitable public evidence found; or claim is opinion/ambiguous.

Confidence (0–1): map from evidence quality/quantity/recency/consistency (e.g., CH hard match → +0.4; each independent reputable source, recent (<12m) → +0.15; minor mismatch −0.1; contradictions −0.3; cap 1.0).

9) Operational constraints & safeguards

Respect robots.txt and Terms of Service.

Rate limiting & caching: obey NewsAPI plan limits; cache CH lookups per company number. 
News API

Deduplicate near-identical claims; cluster by entity + predicate + time.

Time normalisation: convert dates to ISO 8601; align article publish dates to UTC.

Source quality: prefer established outlets in NewsAPI results; down-weight low-credibility domains.

Safety: do not output personal data beyond what public sources provide; clearly label forecasts and marketing language.

10) Acceptance criteria (MVP)

Given a typical UK company “About” page, the tool extracts ≥10 atomic claims, resolves the primary company in CH, and returns verdicts for ≥70% of claims within one run.

“Company status”, “incorporation date”, and “registered office town” claims are correctly verified against CH for that entity.

News-based event claims show at least two corroborating articles when labelled Verified.

The report renders a clear per-claim row with verdict, confidence, and links.
