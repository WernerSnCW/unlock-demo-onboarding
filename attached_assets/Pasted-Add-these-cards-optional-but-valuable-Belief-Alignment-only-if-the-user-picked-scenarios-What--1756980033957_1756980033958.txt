Add these cards (optional but valuable)

Belief Alignment (only if the user picked scenarios)

What to show: Belief alignment: X → Y (↑ +Δ)

How:

Blend the scenario priors into a single vector S.

L1 distances: d_now = Σ|current - S|, d_tgt = Σ|target - S|.

Score: align = 100 × (1 − d/2) → alignmentNow, alignmentTarget.

If alignmentTarget > alignmentNow, show the green “+Δ”.

Diversification

What to show: Diversification: +ΔHHI (lower is better) and/or Effective N: N_now → N_tgt

How:

HHI_now = Σ w_now^2, HHI_tgt = Σ w_tgt^2, ΔHHI = HHI_now − HHI_tgt (positive = improvement).

N_eff = 1 / HHI.

Turnover (and optional cost)

What to show: Turnover: T pp and (if you have friction rates) Est. cost: ~c% of portfolio.

How:

T = 0.5 × Σ|Δ| (in percentage points).

Cost (if you have a per-bucket round-trip rate f_b): cost = Σ (|Δ_b|/2 × f_b).

Liquidity Runway (only if you know monthly spend & portfolio value)

What to show: Liquidity runway: m_now → m_tgt months.

How:

cash_like = (CASH + BILLS_SHORT_GILTS)

runway = (cash_like × portfolioValue) / monthlySpend.

Scenario sanity chip (only if scenarios provided)

What to show (if triggered):

S007: keep real-asset hedge (commodities + gold) when target would reduce it and current <35%.

S009: don’t increase duration (gilts long) when target > current.
