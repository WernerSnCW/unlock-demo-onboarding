PROJECT CONTEXT
We have a new /onboarding-v2 wizard with steps like Welcome, Method, Intake, Holdings, Beliefs, Analysis, Target, Transition, Wrappers, Report.

The next milestone is to implement the real analysis maths that:

Loads numeric policy/guardrail parameters from a config file.

Computes Safety Lights (liquidity, concentration, illiquids) from investor intake.

Enforces “any Red ⇒ tilts disabled”.

Exposes this via a backend API endpoint that the v2 wizard can call.

Uses a small set of golden cases as tests.

You should create the necessary config/test files inside the repo as part of this task, since they don’t exist yet.

STEP 1 – Create a policy defaults file

Create a policy config file in a sensible place, for example config/policy_defaults.yaml or config/14_POLICY_DEFAULTS.yaml. Use YAML format.

Give it a clear structure with sections. At minimum include:

projection (for cash runway and concentration)

collectibles (for illiquids / collectibles limits)

factors (for beliefs → tilts later)

cgt (for CGT pacing later)

wrappers (for wrapper priorities and Bed & ISA)

Populate these sections with explicit numeric defaults (these are policy, not hard-coded in code). Use these values:

Under projection:

min_cash_months: 6

cash_amber_multiple: 1.5

(i.e. runway < 6 months = Red, 6–9 months = Amber, ≥ 9 months = Green).

max_single_name_pct: 0.20 (20% cap for any single holding).

concentration_amber_fraction: 0.75

(i.e. up to 15% is Green, 15–20% is Amber, above 20% is Red).

Under collectibles:

max_weight_pct: 0.10 (10% cap for illiquids/collectibles).

amber_fraction: 0.7

(i.e. up to 7% Green, 7–10% Amber, above 10% Red).

Under factors (for later use in beliefs → tilts maths):

z_cap: 2.0

tilt_budget_risk: 0.30 (for example, at most 30% of current portfolio volatility can be used for tilts).

Under cgt (for later transition planning):

cgt_allowance_per_year_gbp: 3000 (current UK CGT allowance, can be tuned later).

max_annual_disposal_ratio: 0.25 (e.g. sell at most 25% of an oversized position per year).

min_reduce_plan_years: 3 (minimum number of years over which to spread a large reduction).

Under wrappers:

priority_order: a list, in order of preference, like ["ISA", "SIPP", "GIA"].

bed_and_isa with:

min_gain_trigger_gbp: 1000 (only suggest Bed & ISA moves if unrealised gains exceed this).

Add comments in the YAML file explaining each parameter in plain English so that future editors understand what they are tuning.

STEP 4 – Safety Lights analysis service

Create or extend an analysis service module, for example services/analysis.ts (or similar). This will implement the Safety Lights maths and later other onboarding algorithms.

Implement a function (name it appropriately) that:

Takes as input:

An intake or summary object with at least:

cash (liquid cash / emergency fund, in GBP)

spend (annual spend in GBP, or monthly if you prefer, but be consistent)

largest_line_pct (fraction 0–1)

illiquid_pct (fraction 0–1)

A policy object from getPolicy().

Computes cash runway in months:

If spend is annual, use cash_months = cash / (spend / 12).

Handle spend <= 0 robustly (for example, treat runway as a very large number or clamp to a high maximum).

Classifies liquidity:

Red if cash_months < min_cash_months.

Amber if min_cash_months ≤ cash_months < min_cash_months * cash_amber_multiple.

Green otherwise.

Classifies concentration using largest_line_pct:

Use max_single_name_pct as the Red cap.

Use concentration_amber_fraction * max_single_name_pct as the Amber threshold.

Below Amber threshold = Green; between Amber and cap = Amber; above cap = Red.

Classifies illiquids / collectibles using illiquid_pct:

Red if illiquid_pct > max_weight_pct.

Amber if between amber_fraction * max_weight_pct and max_weight_pct.

Green below that.

Combines these into a result object with at least:

liquidity: "GREEN" | "AMBER" | "RED"

concentration: "GREEN" | "AMBER" | "RED"

illiquids: "GREEN" | "AMBER" | "RED"

Optionally overall_status (for example, the worst of the three).

tilts_allowed: boolean which is false if any of the three are Red, otherwise true.

Ensure the maths is written cleanly and uses the policy loader for all thresholds, not hard-coded numbers.

Add brief comments referencing the business rules:

Cash runway rule.

Concentration cap and Amber zone.

Illiquids cap and Amber zone.

“Any Red ⇒ tilts disabled”.
